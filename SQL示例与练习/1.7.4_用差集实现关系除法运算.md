## 问题:
虽然SQL标准化了集合运算的加(union)/减(except)/乘(cross join), 但是还没有标准化
除法运算, 所以为了实现除法运算必须自己实现.
通常有如下几个方法:
* 嵌套使用Not EXISTS;
* 使用HAVING子句转换为一对一关系;
* 把除法变成加法;

## 解法:
使用将除法转换为减法:
```sql
select distinct emp
    from emp_skills es1
where not exists(
    select skill
        from skills
    except
    select skill
        from emp_skills es2
    where es1.emp = es2.emp);
```
🤓这个解法的思路很像面向对象的思路, 逐个遍历所有的员工, 然后检测每个员工是否满足条件.
这是因为关联子查询本身就是为了实现面向对象的循环而设计的啊.
